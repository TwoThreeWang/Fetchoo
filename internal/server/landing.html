Fetch the web, cleanly.

Web content, ready for AI. 多源降级抓取，JSON 结构化输出，智能缓存。

Try API GitHub

串行降级抓取

HTTP → Browser → WebPageSnap → markdown.new → Defuddle → Jina → Brave Search，8 级智能降级，失败自动兜底。

多源聚合

8 种抓取策略：HTTP、Browser、WebPageSnap、markdown.new、Defuddle、Jina、FxTwitter（Twitter 专用）、Brave Search（搜索兜底）。

智能缓存

内存 LRU + SQLite 双级缓存，TTL 过期自动清理，重复请求毫秒级响应。

API 演示

GET /fetch?url=...
// Request
curl
"https://fetchoo.c2v2.com/fetch?url=https://example.com"
// Response

  "data": {

  }
}

响应字段说明

字段 类型 说明
code int HTTP 状态码（200 成功，400 参数错误，500 服务端错误）
message string 状态描述
data.url string 目标网页 URL（可能被 canonical 修正）
data.title string 页面标题
data.content string 提取的正文内容（Markdown 或纯文本）
data.mode string 抓取模式：http / stealth / webpagesnap / markdown-new / defuddle / jina / fxtwitter / brave-search / cached
data.fetch_time float 抓取耗时（秒）
data.content_length int content 字符数
data.selector_matched string 匹配到的 CSS 选择器（如 article、main 等）
data.success bool 是否成功抓取到有效内容
data.metadata.* object 页面元信息：title / description / image / author / publish_date / site_name
GET /fetch

?url= 必需 | ?stealth= | ?no_cache=
POST /batch-fetch
批量抓取（最多 10 并发）
